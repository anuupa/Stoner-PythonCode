cfg_model_from_ini¶

Stoner.Fit.cfg_model_from_ini(inifile, model=None, data=None)[source]¶

Utility function to configure an lmfit Model from an inifile.

Parameters:

inifile (str or file) – Path to the ini file to be read.
Keyword Arguments:
 
  • model (str, callable, lmfit.Model instance or sub-class or None) – What to use as a model function.
  • data (DataFile) – if supplied, the details of the parameter hints and labels and units are included in the data’s metadata.
Returns:

An llmfit.Model,, a 2D array of starting values for each parameter

model can be of several different types that determine what to do:

  • A string. In which ase it should be a fully qualified name of a function or class to be imported. The part after the final period will be assumed to be the name and the remainder the module to be imported.
  • A callable object. In this case the callable will be passed to the constructor of Model and a fresh Model instance is constructed
  • A subclass of lmfit.Model - in whcih case it is instantiated.
  • A Model instance - in which case no further action is necessary.

The returned model is configured with parameter hints for fitting with. The second return value is a 2D array which lists the starting values for one or more fits. If the inifile describes mapping out the \(\Chi^2\) as a function of the parameters, then this array has a separate row for each iteration.
